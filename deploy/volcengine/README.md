# 火山引擎部署（ECS + Docker）

此目录提供一个无状态的首发部署路径：火山引擎 ECS 运行 Docker Compose，Caddy 负责 TLS 与反向代理，Node API 负责排盘和五个场景的规则计算。

## 为什么首发不接数据库

当前 API 不持久化出生资料，也不记录请求体。它在降低个人资料暴露面的同时，仍可支撑建档后实时排盘、今日建议、穿衣、出行、工作推进、选时间与座位娱乐参考。

用户希望跨设备保存命盘、历史或订阅时，再单独接入数据库；那一步必须先完成微信登录换取用户内部 ID、字段加密、数据保留期限和删除接口。

## 部署前置条件

1. 已有火山引擎 ECS、可解析到该实例的域名，且域名备案完成。
2. 安全组仅开放 80/443；不要暴露 API 的 8080 端口。
3. ECS 已安装 Docker 与 Docker Compose Plugin。
4. 在微信小程序后台将 `https://<API_DOMAIN>` 配为服务器域名。

## 部署步骤

```bash
cd deploy/volcengine
cp .env.example .env
# 编辑 .env，将 API_DOMAIN 改为已备案的实际域名
docker compose -f compose.prod.yml up -d --build
curl https://<API_DOMAIN>/healthz
```

健康检查应返回 `{ "ok": true, ... }`。Caddy 会在域名正确解析且 80/443 可访问时自动申请和续期 TLS 证书。

## 上线后的必要设置

- 小程序只调用 `/v1/readings` 与 `/v1/rulebook`；请求的超时建议设为 5 秒。
- 开启 ECS 监控、访问日志与告警，但严禁把请求体、出生日期、时辰或地点写入日志。
- `APP_SECRET`、火山引擎密钥和微信密钥只放在服务端密钥管理中，不进入小程序包、Git 或 `.env.example`。
- 当接入账户和存储后，使用数据库私网连接、字段级加密、删除任务和可审计的规则版本表。

## 不要做的事

- 不在开发者工具里关闭合法域名校验后就当作完成发布验证。
- 不将 API 或数据库直接暴露在公网端口。
- 不以调试日志记录出生资料或整段命理解读。
