# AI 算命小助手｜产品计划

## 0. 产品一句话

一个以子平八字为推演内核、面向日常决策的命理助手：先给出可追溯的命理判断，再将喜忌映射为衣着、方位、行程、沟通等轻量建议。

它不是“预测并保证结果”的工具；它帮助用户在不确定的日常里，获得一个有传统命理依据、也保留现实判断空间的参考视角。

## 1. 北极星与首版边界

### 北极星体验

用户在十秒内可以问出一个具体问题，例如“今天穿什么颜色去见客户？”；在三十秒内得到：

1. 一个简洁可执行的建议；
2. 一个可替代方案（受现实条件限制时仍可用）；
3. 一段清楚的命理依据；
4. 一个不夸大确定性的现实提醒。

### v1 要做

- 八字基础排盘：四柱、藏干、十神、五行、节气月、强弱初判。
- 用神/喜忌的规则化推演与可读解释。
- 今日流日、流时与本命盘的交互判断。
- 三个高频场景：穿衣颜色、出门/办事方位、工作或社交行动建议。
- “问一问”入口：将自然语言路由到已有场景，并允许补充时间、地点、可选项。
- 命理依据页：展示简化后的推演链，而非黑箱结论。
- 用户资料、隐私控制和免责声明。

### v1 暂不做

- 合婚、择日、改名、风水勘测等需要独立体系的能力。
- 以神煞为主导的判断；神煞只可作弱提示。
- “保证赢钱”“化解灾祸”“医疗、投资、法律结论”等承诺或诱导。
- 自动不断追问、过度依赖聊天记录的情感绑定设计。

## 2. 用户与核心任务

| 用户 | 当下任务 | 期待得到什么 |
| --- | --- | --- |
| 好奇的新用户 | 看看今天是否有可用建议 | 低门槛、没术语压迫感 |
| 相信八字的用户 | 验证算法是否“讲得通” | 月令、用神、流日等推演依据 |
| 有具体事件的人 | 为会面、出行、穿衣做选择 | 可执行的优先/备选方案 |
| 深度用户 | 检查自己的命盘与当前周期 | 足够透明、可讨论、可反馈 |

## 3. 产品结构

```text
启动 / 建档
  └─ 首页：今日
      ├─ 今日建议（颜色、方位、行动）
      ├─ 场景快捷入口
      └─ 问一问
  ├─ 场景结果页
      ├─ 结论 / 备选 / 注意
      └─ 为什么这么说
  ├─ 命理依据页
      ├─ 本命盘摘要
      ├─ 今日时空信息
      └─ 推演链
  └─ 我的
      ├─ 出生资料与命盘
      ├─ 反馈记录
      └─ 隐私与说明
```

## 4. 页面与交互计划

### 4.1 启动与建档

**目标：** 在首次使用中建立足够准确的基础资料，不让填写表单破坏仪式感。

- 第 1 屏：一句平静的说明——“用传统命理，为日常选择多提供一个角度。”
- 第 2 屏：出生日期、出生时段（未知可选）、出生地、性别（可跳过，说明仅用于某些传统大运规则）。
- 第 3 屏：资料核对，展示“将按节气换月、当地时区排盘”。
- 完成页：先呈现一句命盘摘要，再进入首页；不在 onboarding 堆满八字术语。

### 4.2 首页｜今日

**目标：** 一眼知道“今天该怎么用它”。

- 顶部仅放日期、节气/干支与一句当天主题。
- 主区是一个“今日建议”模块：一个主建议、两个小建议；不是三张相同视觉权重的卡片。
- 场景入口以横向文字按钮或图标+短标题呈现：`穿什么`、`去哪里`、`怎么推进`、`问一问`。
- 底部有“查看依据”，而非把密集术语塞进首页。

### 4.3 场景提问与结果

**目标：** 让抽象的喜忌变成现实选择。

- 先问场景必需条件。例如衣着场景可选“正式 / 日常 / 运动”；出门可选目的和大致时间。
- 结果固定为四段：`结论`、`备选`、`原因`、`现实提醒`。
- 用户提供选项时（如四个座位、衣柜里的颜色），系统应对具体选项排序，而非泛泛给方位或色系。
- 每次结果可反馈“符合 / 不太适用”，并可标注原因：场地限制、时间不准、建议太泛、其他。

### 4.4 命理依据

**目标：** 让熟悉命理的用户检查推演，让普通用户能理解核心理由。

- 默认展示：日主、月令、强弱倾向、用神、喜忌、今日干支。
- 点击再展开：藏干、十神、合冲刑害、调候、流时变化。
- 用“事实 → 判断 → 对场景的影响”的顺序表达。
- 每个结论都带规则 ID 与版本，不直接长篇复制古籍。

### 4.5 我的

- 可编辑出生资料；修改后让用户明确确认重新计算。
- 可删除命盘、问答与反馈记录。
- 清楚说明：命理推演仅作文化娱乐与自我探索，不应用于医疗、法律、投资或赌博决定。

## 5. 审美与界面系统

以下规则是产品验收条件，不是软建议。

1. **标题采用细线条字体。** 中文优先选择有轻字重的现代无衬线或宋体风格字体；大标题用 Light/300–400，避免粗黑大字和“玄学海报感”。
2. **不要 Card 套 Card。** 一个页面只允许一个主要内容容器层级；内部用留白、细分割线、底色区块或排版建立层次。禁止外层圆角卡片中再塞一组圆角卡片。
3. **克制使用颜色。** 基础为暖白/墨黑/雾灰；每个页面只用一个强调色，强调色来自当天五行但必须降饱和。不要同时出现五行五色、渐变、金箔、霓虹。
4. **留白比装饰更重要。** 采用较大的上下间距、短段落和清晰字级；信息密度高的依据页使用栅格与分割线，不用装饰边框。
5. **用材质而非符号制造气质。** 可用极淡纸张纹理、细线、节气印记；避免八卦、罗盘、龙凤、燃烧符纸等默认“国潮玄学”符号。
6. **动效应像呼吸，不像开奖。** 页面过渡 180–240ms，结果按段淡入；禁止抖动、闪烁、转盘式“命运揭晓”特效。
7. **语言克制。** 不用“天机不可泄露”“必有大劫”一类恐吓式措辞；结论的语气温和、具体、可撤回。

| 用途 | 建议 |
| --- | --- |
| 背景 | `#F7F4EE` 暖白 |
| 主要文字 | `#252522` 墨黑 |
| 次要文字/线 | `#7E7A72` / `#D9D4CA` |
| 默认强调 | `#526B63` 灰青 |

## 6. 分阶段路线

### P0｜规则与内容原型（1–2 周）

- 明确子平规则优先级与冲突处理方式。
- 用 30–50 组匿名样本做人工验盘和推演审阅。
- 为每条规则建立 ID、条件、结果、来源定位、版本与测试用例。
- 先手工完成三个场景的映射表，不急于接入开放式 AI。

### P1｜可用 MVP（3–5 周）

- 完成排盘、喜忌推演、日时计算和三个场景。
- 实现建档、今日页、场景结果、依据页、资料页。
- AI 仅做受控表达：只能引用结构化推演结果，不能改写事实。
- 小范围内测，收集“推演是否自洽”和“建议是否有用”两个维度的反馈。

### P2｜增强个性化（2–4 周）

- 支持时间/地点/现实选项的评分式建议。
- 支持会面、出行、沟通、日程安排等新场景。
- 依据页提供规则版本、简化/专业两种阅读模式。
- 基于反馈改善场景可执行性，不使用“命中率”误导性宣传。

### P3｜专业化与规模验证

- 建立命理师审阅后台、规则灰度发布、争议案例标注。
- 评估真太阳时、不同流派规则配置等高级能力。
- 做隐私审计与内容安全审计，再考虑订阅、报告等商业化。

## 7. 验收指标

- 规则：每个线上结论可回溯到输入、规则版本和中间推演结果。
- 体验：新用户在 3 分钟内完成建档并得到第一条场景建议。
- 清晰：测试用户能说出系统建议的“一个理由”和“一个现实限制”。
- 风格：首页同屏强调色不超过 1 个；任何主要页面没有超过一层的嵌套卡片。
- 安全：涉及疾病、财务下注、违法、伤害等高风险提问时，拒绝做确定性占断并提供现实支持建议。
